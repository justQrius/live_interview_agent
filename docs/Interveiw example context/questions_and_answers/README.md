## How to Use These Files

### For Interview Preparation

**1. Initial Study (Week Before Interview)**:
- Read all files once to internalize narratives
- Practice "Tell Me About Yourself" out loud until natural
- Memorize core framing statements (the one-liners that open each answer)

**2. Deep Preparation (2-3 Days Before)**:
- Pick 3-4 stories you're most confident in
- Practice 30-second, 2-minute, and 5-minute versions
- Anticipate follow-up questions and prepare pivots

**3. Day-Of Preparation**:
- Review core framing statements only
- Read "motivation_and_fit.md" to refresh on "Why Smarsh"
- Review questions to ask Dominic

### For Live Interview Agent (RAG Consumption)

These files are optimized for LLM retrieval:
- **Modular structure**: Each file is self-contained
- **Clear headers**: Questions clearly marked for easy matching
- **Multiple versions**: 30-second, 2-minute, and full versions available
- **Context included**: Each answer includes "Interviewer Intent" to help agent understand *why* the question is asked

### During Interview

**If Question Matches Exactly**:
Use the canonical answer structure, but speak naturally (don't recite verbatim)

**If Question Is Similar**:
Use the core framing statement and adapt the story to fit

**If Follow-Up Question**:
Check the "If Dominic Pushes" sections for prepared responses

## Answer Framework (Common Pattern)

Most answers follow this structure:

1. **Core Framing** (1 sentence): Sets tone and positions you immediately
2. **Canonical Story** (2-5 minutes): Full STAR-formatted narrative
3. **What You're Demonstrating** (Internal note): Signals you're sending without saying explicitly
4. **30-Second Version**: For time-constrained situations
5. **Follow-Up Ready**: Anticipated probes and prepared pivots
6. **Traps to Avoid**: Common mistakes explicitly called out

## Key Themes Across All Answers

### Consistent Messages

1. **Trust Over Features**: Always position solutions as risk mitigation and trust-building
2. **Honesty Without Loss of Authority**: Comfortable admitting gaps while maintaining credibility
3. **Operational Outcomes Over Technical Cleverness**: Focus on business impact, not architecture theatrics
4. **Regulated Environment Fluency**: Demonstrates understanding of compliance mindset
5. **Mature Presales Thinking**: Shows evolution from "feature seller" to "trusted advisor"

### Tone & Style

- **Confident Without Arrogance**: Competent but collaborative
- **Specific Without Rambling**: Concrete examples, tight narratives
- **Technical Without Jargon**: Depth when needed, simplicity when appropriate
- **Consultative Without Passivity**: Challenge assumptions respectfully
