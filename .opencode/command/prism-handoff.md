# Prism Handoff - Session Transition Protocol

This command creates a formal handoff document before ending a session, ensuring context is preserved for the next session (whether yours or a teammate's).

## Why This Matters

From the article: "Experience builds judgment and taste." A proper handoff captures not just WHAT was done, but WHY decisions were made - the judgment that's often lost when context is cleared.

## Usage

```
/prism-handoff              # Standard handoff to session notes
/prism-handoff new-session  # Preparing for /compact or fresh start
/prism-handoff teammate     # Handoff to another person
/prism-handoff tomorrow     # End-of-day handoff to yourself
```

---

## Phase 1: Gather Context

**Actions**:
1. Read `_prism/status.yaml` for current phase
2. Read `_prism/todo.md` for pending items
3. Check git status for uncommitted changes
4. Check beads for in-progress items: `bd list --status in_progress`
5. Review recent conversation for key decisions
6. Run `/prism-reflect` to capture session learnings

---

## Phase 2: Generate Handoff Document

Create/update `_prism/session-notes.md` with structured handoff:

```markdown
# Session Handoff - [timestamp]

## Session Summary
[2-3 sentence summary of what was accomplished]

## Current State

### Phase: [Planning|Solutioning|Implementation|Verification]

### In Progress
- [task 1] - [status/blockers]
- [task 2] - [status/blockers]

### Just Completed
- [completed item 1]
- [completed item 2]

## Key Decisions Made
| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| [decision] | [why] | [what else was considered] |

## Pending Todos (from todo.md)
- [ ] [todo 1]
- [ ] [todo 2]

## Context for Next Session

### What to Know
- [critical context that might be lost]
- [non-obvious state or configuration]

### What to Do First
1. [immediate next step]
2. [follow-up action]

### Potential Gotchas
- [thing that might cause confusion]
- [state that's not obvious from code]

## Uncommitted Changes
[git status summary or "All changes committed"]

## Files Modified This Session
[list of key files touched]

---
*Handoff generated by /prism-handoff at [timestamp]*
```

---

## Phase 3: Handoff-Specific Actions

### For new-session
1. Generate handoff document
2. Suggest: "Ready for /compact. After compaction, run `/prism-continue` to restore context."

### For teammate
1. Generate handoff document
2. Add "## For the Next Developer" section with:
   - Environment setup notes
   - Any local-only state
   - Credentials/config needed

### For tomorrow
1. Generate handoff document
2. Add "## Tomorrow's Priorities" section
3. Note any time-sensitive items

---

## Phase 4: Confirm and Advise

After generating handoff:

```
✅ Handoff document created at _prism/session-notes.md

Next steps:
- Review the handoff for accuracy
- Commit if you want it in git: git add _prism/session-notes.md && git commit -m "chore: session handoff"
- Ready to: [/compact | start new session | end for today]

Context preserved:
- Current phase: [phase]
- Pending todos: [count]
- Key decisions: [count]
```

## Companion Commands

| Command | When to Use |
|---------|-------------|
| `/prism-handoff` | Before clearing context |
| `/prism-continue` | After starting new session (reads handoff) |
| `/prism-todo` | Throughout session for attention anchoring |

---

## Best Practices

1. **Handoff at 60% context** - Before performance degrades
2. **Include the "why"** - Decisions without rationale are lost knowledge
3. **Note the non-obvious** - What would confuse you tomorrow?
4. **Commit important handoffs** - Git preserves across sessions
5. **Read handoff first thing** - Restore context before diving in
